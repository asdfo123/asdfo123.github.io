ForgeWM

A Reproducible Training Recipe for Action-Controllable World Models

Xinye Li

Code Models Data Paper (coming soon)

Abstract

We present ForgeWM, the first fully open and reproducible training recipe for building real-time, action-controllable world models from the Matrix-Game 2 lineage. ForgeWM integrates MG2's game-native I2V backbone, GameFactory's open Minecraft gameplay data, and the Causal Forcing distillation paradigm into a 4-stage progressive pipeline trainable on 8 GPUs. Our final model achieves comparable generation quality to MG2's proprietary distilled checkpoint at 4-step inference, while fixing key failure modes (underwater drift artifacts) and providing complete training code, data, and checkpoints for the community.

ForgeWM generates interactive Minecraft video conditioned on keyboard and mouse inputs, running at real-time speed with 4-step inference.

Key Contributions

Fully Reproducible

Complete training code, pre-encoded data (89 GB LMDB), and checkpoints (Stage 0 + Stage 3) are publicly available.

4-Stage Distillation

Bidirectional SFT → Teacher-Forcing AR → Consistency Distillation → DMD, building on Causal Forcing.

Game-Native Control

Discrete keyboard (WASD) + continuous mouse input, injected via cross-attention and channel concatenation.

Open Data

Trained entirely on GameFactory's GF-Minecraft (~70h gameplay), with scripts to reproduce data preparation.

Architecture

I2V Conditioning (First-Frame Fidelity)

ForgeWM uses a three-pathway image conditioning mechanism inherited from Matrix-Game 2:

  1. Channel-concat: The first frame is VAE-encoded and concatenated channel-wise with the noise input (4-ch mask | 16-ch img_latent = 20 channels). A binary mask marks frame 0 as "real" and subsequent frames as "to generate".
  2. CLIP visual context: The first frame is encoded through a CLIP vision encoder into a 257-token sequence, injected via cross-attention at every transformer block for high-level semantic guidance.
  3. Causal history: Previously generated clean frames are cached in the KV store during autoregressive rollout.

Action Injection

  • Keyboard (discrete): Cross-attention injection — keyboard actions are embedded and attend to latent frame tokens at each transformer block.
  • Mouse (continuous): Sliding-window grouping (VAE temporal compression = 4) followed by channel concatenation with latent features.

Temporal Architecture

  • Block-wise causal attention: Frames grouped into chunks of 3; within-chunk bidirectional, cross-chunk strictly causal.
  • Sliding window (size=6): Each chunk attends only to the 6 most recent frames, enabling unbounded-length generation without memory growth.

Observations

  • Overall quality: ForgeWM largely reproduces MG2's generation quality. Temporal smoothness is slightly better; fine-grained texture detail is slightly weaker (smaller training data: ~70h vs MG2's proprietary dataset).
  • "Underwater" artifact fixed: MG2 tends to drift into underwater/ocean textures when encountering rain, blue sky, or dark scenes — likely from over-representation of ocean footage in its proprietary data. ForgeWM does not exhibit this failure mode.
  • Action controllability: Both models respond correctly to keyboard/mouse inputs. Causal Forcing distillation preserves action fidelity through all 4 stages.
  • HUD stability: MG2's HUD elements gradually shrink over a rollout (possible resolution mismatch). ForgeWM does not show this artifact.
